# Go

Go (also known as Golang) is a statically typed, compiled language designed for systems programming, network services, and CLI tooling. In the Forgejo-Mind stack, Go is relevant because Forgejo itself is written in Go, several CI tooling utilities are Go programs, and the Forgejo-Mind automation codebase may include Go-based microservices and CLI tools. Installing Go from the official upstream tarball (rather than the Ubuntu APT package, which lags several minor versions behind) ensures access to the latest language features, standard library improvements, and performance optimisations. This guide covers downloading, installing, and verifying the Go toolchain.

## Prerequisites

- [Ubuntu 24.04 LTS](01-ubuntu-24-lts.md) — the host OS must be installed and updated.

## Installation

### 1. Check the Current Stable Release

Visit https://go.dev/dl/ to determine the current stable release. At the time of writing this guide, the current version is `go1.24.3`. Replace `1.24.3` in all commands below with the actual current version.

```bash
# Check the Ubuntu APT version (for comparison — usually older)
apt-cache show golang-go 2>/dev/null | grep Version | head -1

# The upstream Go release page always has the latest:
# https://go.dev/dl/
```

### 2. Download the Go Tarball

```bash
GO_VERSION="1.24.3"
GO_TARBALL="go${GO_VERSION}.linux-amd64.tar.gz"
GO_URL="https://go.dev/dl/${GO_TARBALL}"
GO_SHA_URL="https://dl.google.com/go/${GO_TARBALL}.sha256"

# Download the tarball
wget -O "${HOME}/${GO_TARBALL}" "${GO_URL}"

# Download the SHA256 checksum
wget -O "${HOME}/${GO_TARBALL}.sha256" "${GO_SHA_URL}" 2>/dev/null || \
  curl -fsSL "https://go.dev/dl/?mode=json" \
    | python3 -c "
import sys, json
data = json.load(sys.stdin)
for v in data:
    if v['stable']:
        for f in v['files']:
            if 'linux-amd64' in f['filename'] and f['filename'].endswith('.tar.gz'):
                print(f['sha256'])
                break
        break
" > "${HOME}/${GO_TARBALL}.sha256_value"

# Verify the checksum
echo "$(cat ${HOME}/${GO_TARBALL}.sha256_value)  ${HOME}/${GO_TARBALL}" | sha256sum --check
# Expected: OK
```

If the automated checksum download is complex, verify manually at https://go.dev/dl/ by comparing the SHA256 shown on the page with `sha256sum ~/go${GO_VERSION}.linux-amd64.tar.gz`.

### 3. Remove Any Previous Go Installation

If Go was previously installed from another source (APT or a previous tarball), remove it to avoid version conflicts:

```bash
# Remove previous installation if it exists
sudo rm -rf /usr/local/go

# Also remove any Go installed via APT
sudo apt remove --purge golang golang-go golang-1.* 2>/dev/null || true
```

### 4. Extract the Tarball to /usr/local

```bash
sudo tar -C /usr/local -xzf "${HOME}/${GO_TARBALL}"

# Confirm the extraction
ls /usr/local/go/bin/
# Expected: go  gofmt
```

### 5. Add Go to PATH

Add the Go binary directory and the user Go binary directory to the PATH. The user Go binary directory (`$HOME/go/bin`) is where `go install`ed tools are placed.

```bash
# Add to ~/.profile (applies to login shells)
cat >> ~/.profile <<'EOF'

# Go toolchain
export PATH="$PATH:/usr/local/go/bin"
export PATH="$PATH:$HOME/go/bin"
export GOPATH="$HOME/go"
EOF

# Apply immediately in the current shell
export PATH="$PATH:/usr/local/go/bin:$HOME/go/bin"
export GOPATH="$HOME/go"

# If using Zsh, also add to ~/.zshrc
# cat >> ~/.zshrc <<'EOF'
# export PATH="$PATH:/usr/local/go/bin:$HOME/go/bin"
# export GOPATH="$HOME/go"
# EOF
```

### 6. Verify the Installation

```bash
go version
# Expected: go version go1.24.3 linux/amd64
```

### 7. Create a Test Program

```bash
mkdir -p ~/go-test
cat > ~/go-test/hello.go <<'EOF'
package main

import "fmt"

func main() {
    fmt.Println("Go is installed and working!")
}
EOF

cd ~/go-test
go run hello.go
# Expected: Go is installed and working!

# Clean up the test
rm -rf ~/go-test
```

### 8. Clean Up the Downloaded Tarball

```bash
rm -f "${HOME}/${GO_TARBALL}" "${HOME}/${GO_TARBALL}.sha256" "${HOME}/${GO_TARBALL}.sha256_value"
```

## Validation

- [ ] **Go version is correct**

```bash
go version
# Expected: go version go1.24.x linux/amd64
```

- [ ] **GOPATH is set**

```bash
go env GOPATH
# Expected: /home/USERNAME/go
```

- [ ] **GOROOT is set to the installation directory**

```bash
go env GOROOT
# Expected: /usr/local/go
```

- [ ] **go binary is in /usr/local/go/bin**

```bash
which go
# Expected: /usr/local/go/bin/go
```

- [ ] **gofmt is available**

```bash
gofmt --help 2>&1 | head -3
# Expected: usage of gofmt
```

- [ ] **A test program compiles and runs**

```bash
mkdir -p /tmp/go-validation
cat > /tmp/go-validation/main.go <<'EOF'
package main
import "fmt"
func main() { fmt.Println("validation OK") }
EOF
cd /tmp/go-validation
go run main.go
# Expected: validation OK
rm -rf /tmp/go-validation
```

## Deinstallation

```bash
# Step 1: Remove the Go installation directory
sudo rm -rf /usr/local/go

# Step 2: Remove PATH entries from ~/.profile
vim ~/.profile
# Delete the "Go toolchain" section (export PATH ... go/bin lines)

# Step 3: Also remove from ~/.zshrc if present
# vim ~/.zshrc

# Step 4: Remove the GOPATH directory (downloaded packages and installed binaries)
# WARNING: This removes all go install'd tools and the module cache
rm -rf ~/go

# Step 5: Confirm removal
go version
# Expected: command not found (after opening a new terminal)
```

## Continuity Controls

- **Version upgrades:** Go releases a new minor version approximately every 6 months. Upgrading is a simple tarball swap: run `sudo rm -rf /usr/local/go`, download the new tarball, extract it, and verify with `go version`. The module cache in `~/go/pkg/mod` is forward-compatible.
- **Module proxy:** By default, `go get` fetches modules from `proxy.golang.org`. In air-gapped environments, set `GONOSUMCHECK=*` and `GOFLAGS=-mod=vendor` to use vendored dependencies. In the Forgejo-Mind CI environment, configure `GOPROXY=direct` or set up a local Athens module proxy.
- **go.sum verification:** The `go.sum` file cryptographically verifies module integrity. Commit it to the repository alongside `go.mod`. Never delete it unless regenerating from scratch.
- **Tool management:** Use `go install tool@version` to install CLI tools like `golangci-lint`, `buf`, and others into `~/go/bin`. Document the tool versions in a `tools.go` file or a `Makefile` target.
