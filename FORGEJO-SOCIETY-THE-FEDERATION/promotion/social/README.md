# `social/`

> Social posts, threads, banner copy.

This folder holds the **canonical short-form copy for social and
federated channels**. Channel rules live in
[`../04-channels.md`](../04-channels.md); voice rules live in
[`../08-style-guide.md`](../08-style-guide.md).

---

## What belongs here

- **Single posts** — one-shot announcements and observations.
- **Threads** — multi-post sequences explaining a single idea.
- **Banner copy** — short text for profile and channel banners.
- **Reply scripts** — approved short responses to recurring questions
  on social channels.

In line with the project's federation posture, social materials are
written first for **federated** surfaces (Fediverse, sibling forges)
rather than centralised platforms.

## What does not belong here

- Cross-posts to closed platforms the project chooses not to use (see
  `../04-channels.md`).
- Personal posts by maintainers that are not part of project
  promotion.

## File layout (planned)

- `posts.md` — approved single posts, dated.
- `threads/` — one file per thread.
- `banners.md` — approved banner copy.
- `replies.md` — approved reply scripts.

## Status

Scaffold. Files are added as social materials are approved.
