# `coverage/`

> Append-only log of external coverage, with links and notes.

This folder holds the canonical log of **external coverage** of
Forgejo Society. Top-level rules and the entry format live in
[`../05-coverage.md`](../05-coverage.md).

---

## What belongs here

- **Coverage entries**, one per substantive external mention, named
  `coverage-YYYY-MM-DD-publication-short-handle.md`.
- **An index file** (`index.md`), added once the first entry is
  logged, grouped by year.

## What does not belong here

- Social-media reactions and short, ephemeral mentions.
- Private communications.
- Aggregator entries that simply re-link the project's own canonical
  materials.
- The project's *own* announcements or statements (those live in
  [`../announcements/`](../announcements/README.md) and
  [`../statements/`](../statements/README.md)).

## Conventions

- **Append-only.** Coverage entries are not retroactively edited to
  flatter the project. Updates are added as dated follow-ups.
- **Neutral summary first.** The summary describes what was said, not
  how the project feels about it.
- **Link to the canonical.** Where coverage refers to a project
  document, link to that document so readers can compare.
- **Corrections are first-class.** When a correction is requested, the
  request and the response are recorded in the same file.

## Status

Scaffold. The first coverage entry is logged the first time the
project is described in public by someone outside the maintainer
group.
