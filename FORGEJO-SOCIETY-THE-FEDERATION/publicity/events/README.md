# `events/`

> Talk and event records: abstracts, recordings, follow-ups.

This folder holds the canonical record of **public events** at which
Forgejo Society has been represented. Top-level rules and the event
format live in [`../04-events.md`](../04-events.md). The talk
*artefacts* themselves â€” abstracts, slide outlines, demo scripts â€”
live in
[`../../FORGEJO-SOCIETY-PROMOTION/talks/`](../../promotion/talks/README.md).

---

## What belongs here

- **Event records**, one per appearance, named
  `event-YYYY-MM-DD-short-handle.md`.
- **An index file** (`index.md`), added once the first event has
  occurred, listing events in reverse chronological order.

Each event record links out to:

- The talk artefacts in
  [`../../FORGEJO-SOCIETY-PROMOTION/talks/`](../../promotion/talks/README.md).
- Any resulting coverage in [`../coverage/`](../coverage/README.md).
- Any resulting announcement in
  [`../announcements/`](../announcements/README.md).

## What does not belong here

- Slide decks themselves (binary artefacts are linked, not committed).
- Internal calls between maintainers.
- Casual mentions of the project in someone else's talk.

## Conventions

- **One event per file.** A single conference with multiple sessions
  produces multiple files.
- **Honest retrospective.** The follow-up section names what worked
  *and* what did not. We do not edit out failed demos or hard
  questions.
- **Author of record.** Each event record names a human author of
  record, who is also responsible for filing the follow-up.

## Status

Scaffold. The first event record is added when the first public
appearance is scheduled.
