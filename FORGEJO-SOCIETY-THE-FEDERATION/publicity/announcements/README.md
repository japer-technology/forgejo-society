# `announcements/`

> Dated public-facing announcements in the project's own voice.

This folder holds the canonical record of **announcements** issued by
Forgejo Society. Top-level rules and the announcement format live in
[`../02-announcements.md`](../02-announcements.md); voice rules live in
[`../../FORGEJO-SOCIETY-PROMOTION/08-style-guide.md`](../../promotion/08-style-guide.md).

---

## What belongs here

- **Announcement files**, one per announcement, named
  `announcement-YYYY-MM-DD-short-handle.md`.
- **An index file** (`index.md`), added once the first announcement is
  published, listing announcements in reverse chronological order with
  one-line summaries.

## What does not belong here

- Drafts of taglines, narratives, or slogans (those live in
  [`../../FORGEJO-SOCIETY-PROMOTION/messaging/`](../../promotion/messaging/README.md)).
- Internal status updates, sprint notes, or work-in-progress reports.
- Statements responding to external events (those live in
  [`../statements/`](../statements/README.md)).
- Press releases written for distribution to specific outlets (those
  live in
  [`../../FORGEJO-SOCIETY-PROMOTION/press/`](../../promotion/press/README.md)).

## Conventions

- **Append-only.** Announcements are not silently rewritten after
  publication. Material updates are added as dated follow-up sections
  within the same file.
- **One topic per file.** Each announcement covers a single milestone.
- **Linked to substance.** Every claim cites a document under
  `THE-SOCIETY-OF-REPO/`, `FORGEJO-SOCIETY-IMPLEMENTATION/`,
  `FORGEJO-SOCIETY-INSTALLATION/`, or `REPO/forgejo-intelligence/`.

## Status

Scaffold. The first announcement file is added when the first
publicly significant milestone is ready.
