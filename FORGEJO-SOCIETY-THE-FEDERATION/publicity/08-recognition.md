# 08 — Recognition

> How we record awards, mentions, citations, and inclusions.

This document defines how the project records the recognition it
receives. The canonical recognition log lives in
[`recognition/`](recognition/README.md).

---

## What we record

- **Awards** — formal recognition by an organisation.
- **Citations** — citations of the project in academic papers,
  technical reports, books, or standards.
- **Inclusions** — inclusion of the project in curated lists,
  showcases, or directories that have an editorial standard.
- **Notable mentions** — substantive references by recognised people
  or projects in the relevant communities.

We do **not** record:

- Self-nominations or self-curated lists.
- Generic aggregator entries.
- Social-media reactions (those, where worth keeping, are handled in
  [`coverage/`](coverage/README.md)).

## Format

Each recognition entry is a Markdown file in
[`recognition/`](recognition/) named:

```
recognition-YYYY-MM-DD-short-handle.md
```

It contains:

1. **Citation.** Source, awarding body, or citing work, with a stable
   URL or DOI where possible.
2. **What was recognised.** The specific aspect of the project the
   recognition refers to.
3. **Why it matters.** A short, honest note on what the recognition
   does and does not signify for the project.
4. **Linked materials.** Any related announcement, statement, or
   coverage entry.

## Restraint

Recognition is recorded for the historical record, not used as a
billboard. Use of recognition in promotional copy is governed by
[`../FORGEJO-SOCIETY-PROMOTION/08-style-guide.md`](../FORGEJO-SOCIETY-PROMOTION/08-style-guide.md):
specifics over slogans. We do not stack badges; we cite specific,
attributable recognition where it is materially relevant.

## Status

Scaffold. The first recognition entry is added when the first
external recognition arrives.
