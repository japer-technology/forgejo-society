# 09 — Metrics

> What we measure, how we learn, and how promotion feeds the cognitive
> loop.

Promotion is a learning loop, not a broadcast. This document records
**what we measure**, **how we interpret it**, and **how the results
feed back** into the project's cognitive ecology.

---

## What we measure

A small, deliberately narrow set of signals, chosen because they are
honest and easy to verify. To be confirmed and itemised here, with each
metric carrying:

- **Name** — the signal.
- **What it indicates** — and what it does *not* indicate.
- **Source** — where the number comes from (federated peer counts,
  forge stars/forks, talk attendance, press citations).
- **Cadence** — how often it is recorded.
- **Caveats** — known confounds.

Vanity metrics (raw follower counts, impressions divorced from actions)
are explicitly **not** tracked.

## How we interpret

- Movement in a single signal is noise; movement across several is
  evidence.
- Qualitative signals (the kind of person filing issues, the questions
  asked at talks) are weighted alongside quantitative ones.
- We compare each campaign's results against the goal stated in
  `05-campaigns.md`, not against a generic baseline.

## How results feed back

Promotion in this project is part of the same cognitive loop as the
rest of the work. Concretely:

- Each closed campaign produces a short retrospective committed
  alongside the campaign file in this folder.
- Recurring lessons are promoted into updates to the relevant strategy
  document (`01-positioning.md` through `08-style-guide.md`).
- Patterns that recur across campaigns become candidate **K-lines** in
  the Society-of-Repo memory layout
  (`THE-SOCIETY-OF-REPO/02-protocols/06-memory.md`), so that the
  project's mind learns from how it has been described.

## Status

Scaffold. The structure is stable; the metrics list and retrospectives
are populated as campaigns run.
