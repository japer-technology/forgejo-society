# 07 — Press kit

> Press kit contents, project boilerplate, contact protocol, FAQ.

This document defines the **press kit** that journalists, conference
organisers, podcasters, and federated peers can rely on for accurate,
up-to-date material about Forgejo Society.

---

## Boilerplate

A short paragraph that may be quoted verbatim in articles, programme
notes, and federated announcements. Drafted here once approved; until
then, defer to the description in the root `README.md`.

## Fact sheet

A bulleted list of stable facts about the project:

- What it is (one sentence).
- What it runs on (Forgejo, Ubuntu, project-owned hardware).
- Licence (MIT, per the badge in the root `README.md`).
- Where the specification lives (`THE-SOCIETY-OF-REPO/`).
- Where the implementation lives (`.forgejo-society/`,
  `REPO/forgejo-intelligence/`).
- Governance posture (authority levels per
  `THE-SOCIETY-OF-REPO/01-governance/`).
- Federation posture (per
  `FORGEJO-SOCIETY-PLAN/13-inter-repo-communication.md`).

## Quotes

Approved quotations attributable to project maintainers, reviewed and
versioned here so that the same wording can be reused without
re-approval each time.

## Contact protocol

How journalists and external collaborators should contact the project,
who responds, and the expected turnaround. Aligned with governance and
authority levels.

## FAQ

Recurring questions and the project's canonical answers. Examples to be
filled in:

- How is this different from a CI/CD platform?
- How is this different from an agent framework?
- Does it require sending data to external model providers?
- How are agents kept safe and reviewable?
- How does it federate with other societies?

## Asset bundle

A minimal set of assets — logo variants, a hero image, a one-page PDF —
that the press kit references, all sourced from
[`assets/`](assets/README.md). The canonical text content of the
project poster (the source for the one-page PDF) lives in
[`press/poster.md`](press/poster.md).

## Status

Scaffold. Structure is stable; boilerplate, fact sheet, quotes, and FAQ
are populated as the project's public voice is approved.
