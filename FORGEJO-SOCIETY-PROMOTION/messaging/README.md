# `messaging/`

> Reusable copy blocks: taglines, blurbs, abstracts.

This folder holds the **canonical short-form copy** that other surfaces
quote rather than paraphrase. The rules of voice and tone that govern
this copy live in [`../08-style-guide.md`](../08-style-guide.md);
positioning and elevator pitches live in
[`../01-positioning.md`](../01-positioning.md).

---

## What belongs here

- **One-liners** — single-sentence project descriptions for slide
  titles, social banners, and the GitHub repository description.
- **Short blurbs** — 2–3 sentence descriptions for link cards,
  conference programme entries, and federated peer announcements.
- **Long blurbs** — paragraph-length descriptions for the website,
  press kit, and feature articles.
- **Abstracts** — talk and paper abstracts in their canonical form
  (also referenced from [`../talks/`](../talks/README.md)).

## What does not belong here

- Drafts that have not been reviewed (keep in branches or pull
  requests).
- Surface-specific copy that is not reused (keep in the surface that
  uses it).
- Long-form essays (those belong in `FORGEJO-SOCIETY-INTRODUCTION/` or
  `web/` as appropriate).

## File layout (planned)

- `one-liners.md` — list of approved one-liners with intended use.
- `blurbs-short.md` — approved short blurbs.
- `blurbs-long.md` — approved long blurbs.
- `abstracts.md` — approved abstracts, cross-referenced from
  `../talks/`.

## Status

Scaffold. Files listed above are added as the corresponding copy is
approved.
