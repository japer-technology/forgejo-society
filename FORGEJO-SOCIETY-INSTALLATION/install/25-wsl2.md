# Windows Subsystem for Linux 2

Windows Subsystem for Linux 2 (WSL2) runs a real Linux kernel inside a lightweight Hyper-V virtual machine on Windows 10/11, giving Windows developers a full Ubuntu 24.04 LTS environment — including the Linux filesystem, apt package management, and every tool covered in the Linux guides of this library — without a dual-boot setup or separate physical machine. In the Forgejo-Society stack, WSL2 allows Windows developers to work with the same toolchain as the forge server: they can compile Linux binaries, run Linux CI scripts locally, test Forgejo Actions workflows, and use the same shell scripts and Makefiles as the server-side operators. VS Code's WSL Remote extension integrates seamlessly, providing a native editing experience with Linux tooling backing it.

---

## Prerequisites

- Windows 10 version 2004 (build 19041) or later, or Windows 11.
- Virtualisation must be enabled in BIOS/UEFI (Intel VT-x or AMD-V). Most modern systems have this enabled by default.

## Installation

### 1. Enable WSL2 and Install Ubuntu 24.04

Open PowerShell as an administrator:

```powershell
# Install WSL with Ubuntu 24.04 in a single command
# This enables the WSL and Virtual Machine Platform Windows features,
# installs the WSL2 Linux kernel update, and installs Ubuntu 24.04
wsl --install -d Ubuntu-24.04
```

**If prompted to reboot**, do so before continuing:

```powershell
Restart-Computer
```

After the reboot, Windows automatically continues the Ubuntu installation and opens a terminal window prompting you to create a Linux username and password.

### 2. Create the Linux Username and Password

When the Ubuntu 24.04 terminal opens:

```
Enter new UNIX username: yourname
New password: (enter a strong password)
Retype new password: (confirm)
```

This creates a regular Linux user account inside WSL. It is separate from your Windows account.

### 3. Set WSL2 as the Default Version

```powershell
# Ensure new distributions default to WSL2 (should already be set by the installer)
wsl --set-default-version 2

# Verify
wsl --list --verbose
# Expected:
# NAME            STATE           VERSION
# Ubuntu-24.04    Running         2
```

### 4. Update Ubuntu Inside WSL

Switch to the WSL terminal (or open Ubuntu from the Start menu):

```bash
sudo apt update && sudo apt upgrade -y
```

### 5. Follow the Linux Setup Guides Inside WSL

Once WSL2 is running Ubuntu 24.04, follow the Linux guides in this library to set up the full developer toolchain inside WSL:

- Follow [Ubuntu 24.04 LTS](01-ubuntu-24-lts.md) for essential packages, timezone, and SSH hardening (skip the ISO/installer steps — WSL provides the OS).
- Follow [Git and Git LFS (Linux)](13-git-linux.md) to configure Git identity, LFS, and global .gitignore inside WSL.
- Follow the language runtime guides ([Node.js via nvm](15-nodejs-nvm.md), [Go](16-go.md), [Rust](17-rust.md), [Python 3](18-python3.md)) as needed.

```bash
# Install the essential packages from guide 01
sudo apt install -y \
  build-essential pkg-config libssl-dev libffi-dev git curl wget unzip \
  gnupg2 jq yq vim nano tmux htop rsync pv
```

### 6. Install the VS Code WSL Extension

On the Windows side (not inside WSL), install the WSL extension:

```powershell
# In Windows PowerShell
code --install-extension ms-vscode-remote.remote-wsl
```

Now when you run `code .` inside a WSL terminal, VS Code opens in Windows with the WSL remote indicator in the bottom-left corner, and all extensions run on the Linux side.

```bash
# Inside WSL, open a project in VS Code
cd ~/projects/my-project
code .
# VS Code opens in Windows with "WSL: Ubuntu-24.04" in the status bar
```

### 7. Accessing Windows Files from WSL

Windows drives are mounted automatically under `/mnt/`:

```bash
ls /mnt/c/Users/
# Expected: your Windows username directory and other Windows user directories

# Access your Windows documents from WSL
cd /mnt/c/Users/YourWindowsUsername/Documents
ls
```

### 8. Accessing WSL Files from Windows Explorer

The WSL filesystem is accessible in Windows Explorer via the network path `\\wsl$\Ubuntu-24.04\`. You can navigate to it directly:

```powershell
# Open WSL filesystem in Explorer
explorer.exe \\wsl$\Ubuntu-24.04\home\yourname
```

## Validation

- [ ] **WSL2 is installed and Ubuntu 24.04 is the default distro**

```powershell
wsl --list --verbose
# Expected:
#   NAME            STATE           VERSION
# * Ubuntu-24.04    Running         2
```

- [ ] **WSL2 kernel version is current**

```powershell
wsl --version
# Expected: WSL version: x.x.x.x, Kernel version: x.x.x.x, WSLg version: ...
```

- [ ] **Ubuntu is running inside WSL**

```bash
# Inside WSL terminal:
uname -a
# Expected: Linux <hostname> x.x.x-microsoft-standard-WSL2 #1 SMP ... x86_64 GNU/Linux
```

- [ ] **Ubuntu version is correct**

```bash
# Inside WSL terminal:
lsb_release -a
# Expected: Description: Ubuntu 24.04.x LTS
```

- [ ] **Windows filesystem is accessible from WSL**

```bash
# Inside WSL terminal:
ls /mnt/c/Users/
# Expected: Windows user directories are listed
```

- [ ] **WSL filesystem is accessible from Windows**

```powershell
# In Windows PowerShell:
Get-ChildItem "\\wsl$\Ubuntu-24.04\home"
# Expected: your WSL home directory is listed
```

- [ ] **VS Code opens from WSL with WSL remote indicator**

```bash
# Inside WSL terminal:
mkdir -p ~/test-wsl-vscode
cd ~/test-wsl-vscode
git init
code .
# Expected: VS Code opens in Windows with "WSL: Ubuntu-24.04" in bottom-left status bar
rm -rf ~/test-wsl-vscode
```

## Deinstallation

```powershell
# Step 1: Shut down all running WSL instances
wsl --shutdown

# Step 2: Unregister the Ubuntu 24.04 distribution
# WARNING: This permanently deletes the Linux filesystem and all data inside WSL
wsl --unregister Ubuntu-24.04

# Confirm removal
wsl --list --verbose
# Expected: no Ubuntu-24.04 in the list (or "Windows Subsystem for Linux has no installed distributions.")
```

To remove the WSL feature entirely (optional — requires a restart):

```powershell
# Open Windows Features
# Start → search "Turn Windows features on or off"
# Uncheck: "Windows Subsystem for Linux"
# Uncheck: "Virtual Machine Platform"
# Click OK and restart when prompted
```

Or via PowerShell (requires administrator):

```powershell
Disable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux
Disable-WindowsOptionalFeature -Online -FeatureName VirtualMachinePlatform
Restart-Computer
```

## Continuity Controls

- **WSL kernel updates:** WSL2's Linux kernel is updated via Windows Update. Run `wsl --update` to force a kernel update check outside of the normal Windows Update cycle.
- **Backup:** The WSL filesystem is not backed up by Windows Backup by default. Export the WSL distro periodically: `wsl --export Ubuntu-24.04 Ubuntu-24.04-backup.tar`. Store the export on a network drive or external storage.
- **Resource limits:** WSL2 by default uses up to 50% of the host's RAM and all CPU cores. Create a `.wslconfig` file at `%USERPROFILE%\.wslconfig` to set limits:
  ```ini
  [wsl2]
  memory=16GB
  processors=8
  ```
  Restart WSL after changing this file: `wsl --shutdown`.
- **Docker Desktop with WSL2 backend:** If Docker Desktop is installed, it can use the WSL2 backend for better performance. Enable this in Docker Desktop → Settings → General → Use WSL 2 based engine.
